# FAMMS 故障分類體系標準化
## Factory Asset & Maintenance Management System
### Version 1.0

---

## 故障樹設計原則

1. **三層結構**：主分類 → 次分類 → 具體故障碼
2. **明確互斥**：每個故障只能歸到一個葉子節點
3. **支援下拉菜單**：UI 不要求工程師輸入，只選擇
4. **可擴展**：新增故障碼時無需改 DB Schema

## 使用指南

### 對工程師

1. **報修時**：選擇主分類 → 次分類 → 具體故障碼
2. **系統自動記錄**：`failure_code = 'BEARING_001'`
3. **不要手動輸入**——全部是下拉菜單

### 對主管

1. **查看 KPI 時**：可按故障碼、次分類、主分類聚合
2. **範例**：
   - 「軸承故障」(BEARING_*) 佔 35%
   - 「軸承潤滑不足」(BEARING_001) 佔 18%
   - 「感測器」(SENSOR_*) 佔 22%

### 對系統

1. **重複故障判定**：
   ```
   同一設備 + 同一 failure_code + 30天內 + 臨時修復 
   → 系統提示「疑似重複故障」
   ```

2. **RCA 觸發**：
   ```
   同一 failure_code + 90天內 ≥ 3次
   → 必須填寫根因分析
   ```

3. **KPI 計算**：
   ```
   故障分佈、重複率、平均修復時間等
   都按 failure_code 分組計算
   ```

## 故障碼命名規則

**格式**：`[CATEGORY]_[SEQUENCE]`

- `BEARING_001` - 軸承類的第1個故障碼
- `VFD_006` - 變頻器類的第6個故障碼

**優點**：
- 一眼看出大分類
- 易於搜尋和統計
- 易於國際化（代碼不變，標籤翻譯）

## 反複失敗判斷邏輯

### 系統自動檢測

```
事件 A（時間 T1）：
  機器：DIN-HMG-001
  故障碼：BEARING_001（潤滑不足）
  完成類型：Temporary Fix
  → 進入觀察期

事件 B（時間 T2，T2 - T1 ≤ 30天）：
  機器：DIN-HMG-001
  故障碼：BEARING_001（同一故障碼）
  → 系統提示「疑似重複故障」

主管確認：
  ○ 是同一事件（事件 A 延伸，輸入：Temporary Fix 後再發）
  ○ 新增事件（與事件 A 無關）
```

### RCA 觸發條件

```
同一故障碼（如 BEARING_001）
在 90 天內發生 ≥ 3 次

→ 系統強制要求：
  1. Root Cause（根本原因）
  2. Corrective Action（改正措施）
  3. Preventive Action（預防措施）
  4. Responsible Person（負責人）
  5. Due Date（完成期限）
```

## 完成類型（Completion Type）

### Temporary Fix（臨時修復）

風險等級：**高**

範例：
- 重新校正、擰緊螺栓
- 臨時焊接、繞線
- 旁路運轉（bypass）
- 更換保險絲後繼續跑

**系統行為**：
- 30 天內同故障碼再發 → 自動判定 **Repeat Failure**
- 觀察期異常警示

### Permanent Fix（永久修復）

風險等級：**低**

範例：
- 更換馬達、更換軸承
- 更新 PLC 程式
- 更換變頻器、更換感測器
- 完整的根因排除

**系統行為**：
- 後續故障時權重較低
- 不自動判定重複故障
- 但如果 90 天內 ≥3次同故障碼，仍需 RCA
